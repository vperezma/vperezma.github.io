The display property controls the layout. Every element with learned so far in this class has a default display value depending on what type of element is.

A block element is an element that takes up the full width available, and has a line break before and after it.

Examples of block elements:

  • h2
  • p
  • li
  • div

An inline element only takes up as much width as necessary, and does not force line breaks.

Examples of inline elements:

  • span
  • a

div is the standard block-level element. A block-level element starts on a new line and stretches out to the left and right as far as it can. Other common block-level elements are p and form, and new in HTML5 are header, footer, section, and more.

<!-- html code --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Document</title> </head> <body> <section>this is a section</section> </body> </html> <!-- css code --> section{ height:100px; border:10px solid blue; }

span is the standard inline element. An inline element can wrap some text inside a paragraph like this without disrupting the flow of that paragraph. The a element is the most common inline element, since you use them for links.

**Although all of these elements have set display values you can always override them as needed.

<!-- html code --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Document</title> </head> <body> <section>Lorem ipsum dolor sit amet, consectetur adipisicing elit. </section> <span>Lorem ipsum dolor sit amet, consectetur adipisicing elit. </span> </body> </html> <!-- css code --> section{ height:100px; border:10px solid blue; } span{ border: 10px solid green; }

display: inline-block

With inline-block we are telling the browser to display the element inline but allow it to keep its block level status therefore allowing it to create a new row once the browser ends.

<!-- html code --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Document</title> </head> <body> <div class="box2">this is a box Lorem ipsum dolor sit amet. </div> <div class="box2">this is a box Lorem ipsum dolor sit amet. </div> <div class="box2">this is a box Lorem ipsum dolor sit amet. </div> <div class="box2">this is a box Lorem ipsum dolor sit amet. </div> </body> </html> <!-- css code --> .box2{ display: inline-block; width: 200px; height: 100px; margin: 1em; border:10px solid green; }

Let's go over positioning vocabulary

static is the default value. An element with position: static; is not positioned in any special way. A static element is said to be not positioned and an element with its position set to anything else is said to be positioned.

relative behaves the same as static unless you add some extra properties.

Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.

A fixed element is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. As with relative, the top, right, bottom, and left properties are used.
